About this work

Exploring design options for design improvements to the dashboard.

Design 1

Improvements to the search bar

  • During research sessions it users found it difficult to use the search input. This design adds a direct call to action and adds text to advise the user on how they can use this component.

Introducing an information panel

  • Various tenants on DECs require attention to be brought to specific lists or types of cases we have introduced an info panel

Design 2

Exploring using tags as case counts.

  • The current design shows unallocated and overdue cases during user testing sessions user attempted to interact with the the counts. This design adds links to the counts. This would also require a change to DECS to create workstacks that show unallocated or overdue cases.
  • This design also brings tag designs into the dashboard the purpose of using the tags is to improve the use of colour and also to increase familiarity of the tags as they are also used on workstacks. see tags page
